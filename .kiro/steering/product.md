# 製品概要 (Product Overview)

## 製品名
Claude Code ⇄ Claude Desktop Bridge

## 製品の説明
Claude CodeとClaude Desktopの連携を支援するシステム。Claude Codeが実装困難に直面した際、自動的にClaude Desktopに助けを求められる仕組みを提供します。

## コア機能

### 1. ヘルプリクエスト作成
- タイトル、問題の説明、試行内容、関連ファイル、エラーメッセージを構造化
- JSONファイルとして自動保存
- 分析が必要なファイルを自動的にコピー
- Claude Desktopへの指示を自動生成

### 2. レスポンス管理
- Claude Desktopからの回答を確認
- 分析結果と推奨事項をフォーマット表示
- 実装ステップの明確化

### 3. 未回答リクエスト追跡
- 回答待ちのリクエスト一覧表示
- ステータス管理
- 優先順位付け

### 4. アーカイブ機能
- 完了したリクエストとレスポンスを整理
- プロジェクト履歴の保存
- 知識ベースの構築

## ターゲットユースケース

### 主要なシナリオ
1. **パフォーマンス問題のデバッグ**
   - 例: Neo4jクエリが遅い（5000ノードで10秒以上）
   - 例: WAM NETデータ取り込みに5分かかる

2. **設計判断の相談**
   - 例: エコマップのデータモデル設計
   - 例: 複雑なリレーションシップの管理方法

3. **複雑なエラーの解決**
   - 例: 同じエラーが3回以上発生
   - 例: 10分以上進展なし

4. **アーキテクチャの助言**
   - 例: システム全体の影響範囲が不明
   - 例: リファクタリングの方向性

### トリガー条件
以下のいずれかに該当する場合、即座にヘルプリクエストを作成すべき：
- ✅ 同じエラーが3回以上
- ✅ 10分以上進展なし
- ✅ 設計判断が必要
- ✅ 影響範囲が不明

   - パターン認識の向上

2. **コードの質**
   - 専門的な助言による高品質な実装

   - 技術的負債の削減

3. **知識の共有**
   - チーム全体での問題解決パターンの共有
   - アーカイブからの知識再利用
   - 継続的な改善

## 差別化要因

### 他のソリューションとの違い
1. **ファイルベース通信**
   - シンプルで信頼性が高い
   - デバッグが容易
   - 永続化が自動

2. **標準ライブラリのみ**
   - 依存関係なし
   - セットアップが簡単
   - 高い互換性

3. **構造化されたワークフロー**
   - 明確なトリガー条件
   - 標準化されたリクエスト形式
   - 追跡可能な履歴

4. **どのプロジェクトからも利用可能**
   - グローバルな配置
   - sys.path.append()で簡単にインポート
   - 複数プロジェクトでの共有
